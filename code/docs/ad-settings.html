Settings

General - Music Store Name
Enter the name of your music store. Max 100 chars.
General - Email Address
Your main email address. Additional emails can be set via the SMTP tab.
General - Store HTTP Path
The FULL http path to your music store WITH trailing slash. This will be auto created on install.
General - Server Path to Secure Folder
FULL server path to your SECURE folder WITHOUT trailing slash. This is the folder that stores your FULL mp3 files. It should NOT be accessible on the web for security. This is NOT a http web path. Examples of server paths are:

/home/username/SECURE (Linux)
C:\SECURE (Windows)
Once your path is in place, use the "Check" option to see if the location is properly set.
General - Enable Rewrite Functions
This enables the search engine friendly urls. For this to work your server MUST support .htaccess and mod_rewrite.

After enabling this option, rename the 'htaccess_COPY.txt' file to '.htaccess' in your installation.

Your preferred PHP date format. For supported options click here.
Date/Time - Time Format
Your preferred PHP time format. For supported options click here.

Date/Time - Timezone
Default timezone. Dates and times are stored in UTC format and this option correctly displays dates for your timezone.
Date/Time - Calendar Date Format
For the javascript calendar (above screenshot). Dates are auto converted to MySQL format on entry. This option is purely cosmetic.
Store - Default Meta Keywords
Default meta keywords for your store. For search engines.
Store - Default Meta Description
Default meta description for your store. For search engines.
Store - Theme
Your preferred theme. See the themes section for more information.
Store - Email Notification
Set email notification options. Uncheck to disable, check to enable.
Store - Enable CD Purchase Option
Enables or disables the CD purchase option from the store regardless of whether any prices are set. Useful if you only sell downloads. Note that an option to disable/enable the download options is NOT available as this is primarily a download system and downloads would be expected. You can edit the store templates if you don`t want the download option.

Enables or disables the RSS feeds. RSS feeds are an easy way for people to keep up to date with your store content without visiting your site. Its an anonymous subscription, which is very popular. You can read up more on RSS feeds here if you aren`t sure what they are.

In Maian Music, feed links appear for the homepage, styles, latest and popular pages. If enabled you`ll see a link as shown in the example screenshot:

Accounts - Minimum Password Length
Minimum password length for new accounts. People use stupid passwords, so forcing them to enter a minimum amount of characters can help with better passwords.
Accounts - Email Notification
Set email notification options. Uncheck to disable, check to enable.
Accounts - Enable Login Events for New Accounts
Enable or disable login events for new accounts. Can be enabled / disabled per account later.
Accounts - Send Admin Alert Email if More Than XX Login IP Addresses are Detected (Per Account)
This feature can detect account sharing. If set higher than 0, will notify admin if there are XX different login IPs on an account. Note that this should be used cautiously. These days people log in from all over the world on different devices with different IPs. Many also use proxies and VPNs to mask IPs. Set to a conservative limit if used. 0 to disable.
Payment - Payment Mode
Most payment gateways have a sandbox mode enabling you to test the routines without taking live payments from credit cards. This option should ALWAYS be set for your preferred gateways and tested before going live. DO NOT simply set to live and expect it to work. If you have your configuration set wrong it will fail.
Payment - Processing Currency
Your preferred payment processing currency. MUST be supported by your enabled gateways.
Payment - Currency Display
Set how prices display in your store. Use character entities for special characters, NOT the actual character. So, for example for US dollars, enter &#036; instead of simply $. The {AMOUNT} var will parse into the actual price. Here are some examples:

&pound;{AMOUNT} > UK - This would display in your store as: £2.99
&#036;{AMOUNT} > U.S - This would display in your store as: $2.99
{AMOUNT}&euro; > Euro - This would display in your store as: 2.99€
AU&#036;{AMOUNT} > Australian Dollar - This would display in your store as: AU$2.99
HKD&#036;{AMOUNT} > Hong Kong Dollar - This would display in your store as: HKD$2.99
2.99 is just used as an example, not all your prices will be 2.99. Just in case thats what you were thinking. :)
Payment - Next Invoice No
The next invoice number. Usually 1 for a new store, but enter custom number if applicable.
Payment - Min Purchase Amount
If applicable, the min sub total amount allowed before buyers can check out.
Payment - Enable Gateway Log
This should be enabled initially while you are testing the gateways. If you are having problems, refer to the "logs" folder and the gateway logs. Once you are happy everything is working ok, you should remove ALL .txt files from the log folder and disable logging. Some txt files may contain sensitive data relating to your merchant account.
Payment - Process Pending as Completed
In some cases a payment may initially be "Pending". An example would be an echeck payment via Paypal. In a case like this the payment is on hold until the payment clears. If you prefer you can treat this option as completed. It should be enabled with caution as a buyer could cancel the payment after they receive the download activation.

Tax - Default Tax Rate for Tangible Goods (CDs) (%)
The default tax rate for tangible goods if applicable. This is global unless set per country (via the countries section). Enter number ONLY, no percentage sign. For the UK @ 20%, enter 20. Set to 0 to disable fixed rate.

Separating tangible tax from digital tax complies with the EU`s MOSS regulation.
Tax - Default Country for Tangible Tax
The is the default country for tangible tax for tax purposes. If no country overrides are set for the buyers country, this country and the above default tax are used. For example, to comply with MOSS in the UK, you would set the default tax to 20% (current rate at the time of the docs and if you are VAT registered), specify the country as UK and have NO tangible tax overrides in place for countries. This would charge all buyers 20%, regardless of country.
Tax - Default Tax Rate for Digital Goods (Downloads) (%)
The default tax rate for digital goods if applicable. This is global unlessset per country (via the countries section). Enter number ONLY, no percentage sign. For the UK @ 20%, enter 20. Set to 0 to disable fixed rate.

Separating tangible tax from digital tax complies with the EU`s MOSS regulation.
Tax - Default Country for Digital Tax
The is the default country for digital tax for tax purposes. If no country overrides are set for the buyers country, this country and the above default tax are used. For example, to comply with MOSS in the UK, you would set the default tax to 20% (current rate at the time of the docs and if you are VAT registered), specify the country as UK and have E.U ONLY digital tax overrides in place for countries. This would charge all buyers 20%, unless they lived in the E.U, where they would be charged at their countries override rate.
Other > Downloads - Download Links Expire (XX after sale is activated)
If set, download link will expire after given time.
Other > Downloads - Maximum Clicks Per Download Link
If set, only a minimum amount of clicks are allowed per download. This can help reduce bandwidth as many people give friends log in details to download. You can`t do anything about file sharing, but you can keep your server costs down.
Other > Downloads - Lock Sales Page if More Than XX Download IP Addresses are Detected (Per Item)
If set, allows only a certain amount of IP addresses to access a download. Remember that these days people have multiple devices all with different IP addresses, so different IP addresses for the same download can be expected. On saying that it can also enable you to identify people who are sharing downloads. Enable with caution.
Other > Downloads - Send Email Alert to Admin if Page is Locked
Sends email to webmaster if account page is locked. Email details IP addresses and locations.
Other > Downloads - Restrict Downloads to Original Sale IP Address
Strict mode. Enables only the purchase IP to access downloads. Enable with caution. See previous note about multiple devices.
Other > Downloads - Folder for Temporary Zip File Creation
Collections are consolidated into zip files. On zip creation, a temp file is created and on download it is deleted. The server "tmp" directory is recommended for temp files as the servers garbage collector will perform auto cleanups. It should also be the fastest option. The "logs" folder can also be used if you have problems with the server directory.
Other > Downloads - Enable Download Debug Log
Enables the download log. Useful if you have people reporting issues with downloads. It is recommended you refer to this log when testing the system works to make sure downloads are working ok.
Other > SMTP > Host
SMTP host or IP. When using SMTP it is recommended you use a stable SMTP service, like Mandrill or MailGun for mail delivery. This will make your emails more reliable. Any SMTP account can be used. Once your email details are in place use the test button to send a test message to all your accounts. If you still have issues, enable the mail debug log. See F.A.Q

IMPORTANT: If you are using Gmail, DO NOT use your Gmail password. You MUST create an app specific password in your Google account and use that. Also, ONLY use SSL if you have SSL installed. If you don`t, you should use TLS. For the Gmail port ONLY use 465 or 587.
Other > SMTP > Username
SMTP username if authentication is required.
Other > SMTP > Password
SMTP password if authentication is required.
Other > SMTP > Security / Port
SMTP security option and port number. Some services require SSL (such as Google) and others (such as Amazon) TLS.
Other > SMTP > From Name
Your preferred "from" name for outgoing emails.
Other > SMTP > From Email
Your preferred "from" email address for outgoing emails.
Other > SMTP > Additional Notification Email Addresses (CC) (Comma Delimit)
Enter additional emails if you want notifications to go to other addresses besides your main address. Separate with a comma if you are using multiple. Additional emails are sent via Carbon Copy.
Other > Social - AddThis Profile ID (for Social Buttons)
AddThis is a sharing service that enables you to easily put social sharing buttons on your site. Its a FREE service. To enable on collections pages, do the following:

1. Sign up and enter your profile ID in the Maian Music settings box.
2. In the AddThis control panel, click "Tools" and select the free tool "Sharing Buttons". Configure your options, then click activate.
3. Make sure AddThis is enabled for collections in Maian Music via the "Social" tab.
Once enabled you should see the social buttons appear. Here is an example of how your buttons may look:


You can customise what displays via your AddThis control panel. Other paid options are available.
Other > Social - Disqus Comments System - Short Name
Disqus is a FREE comments system. It enables people to comment on your collections. Maybe they have comments about certain tracks. To enable, sign up and enter short name. If you are already using Disqus you may have to add your music domain to the allowed domain list in your Disqus interface:

Admin > Settings > Advanced > Trusted Domains
Everything is managed via the Disqus interface. If you aren`t familiar with Disqus, please take some time to familiarise yourself with the Disqus interface and settings. Its a very powerful comments system.
Other > Social - Disqus Comments System - Category ID
Enter Disqus category ID if applicable.
Other > Social - Pushover User Key / Pushover API Token/Key
Pushover is a notification system that lets you receive notifications to all Pushover enabled devices. For example, you may want to receive noification pings on your phone and not just via standard email. To enable, sign up for a Pushover account and enter your user key and token. You will receive notification when any new sale occurs in Maian Music.

Note: Your user key is shown on the main screen when you log into Pushover, to generate your token click "Apps & Plugins" from the menu.

More information on their API here. To add more options in Maian Music, edit the following file:

control/classes/class.apis.php
The "pushover" function contains the relevant code. Click the button to send a test ping to your devices.
Other > Social - Enable Facebook Open Graph Meta Data
Facebooks Open Graph tags make sharing data on Facebook easier. When someone enters a shop url, the data is pulled from your server and auto filled in on FB. To test, enter a collection url on your timeline and see what happens. After you type a spinner will appear as FB pulls the data. To test before enabling, try an Amazon link.
Other > Social - Default Facebook Open Graph Image (http://)
If someone shares a collection page, the image that would be shown would be the cover art if one exists. If no image is set, facebook will grab something from your page. Therefore you should also set a default image. You can also just modify the default system image which is located at:

content/**THEME**/images/fb-default.png

Other > System Status - Enable System
Enable or disable the system.
Other > System Status - Auto Enable on Date
If disable, set date to auto enable. Clicking into box launches the calendar.
Other > System Status - Offline Reason
Enter reason for system being offline.
Other > System Status - Allow IP Addresses (Comma Delimit)
When the system is offline you can allow access for any IPs. Useful if you want to disable access, but give your developer access. Multiple IPs separated with a comma if applicable.
Other > Shipping Zones
As Maian Music is a download system, shipping options are very basic for CDs. Enter name for shipping zone and rate. Rate can be fixed amount or percentage. ie: 2.99 applies 2.99 charge, 20% applies 20 of sub total. Cick the button to add boxes and the to remove.

To remove specific entry, leave both boxes blank.
Other > Terms & Conditions Text - Enter Terms & Conditions Text
Enter terms & conditions text if this feature is enabled. HTML may be used here, but be careful NOT to break the responsive layout. ONLY use HTML acceptable for a <div> tag.
Other > Terms & Conditions Text - Enable on Checkout
Enable or disable on checkout screen. If enabled a checkbox appears asking the buyer to agree to the terms and conditions. A link to a pop up window will show the text above.
Other > License Agreement Email - Email Subject
This enables you to send out a license agreement email after all music purchases. Useful for legal disclaimers. Enter subject for email.
Other > License Agreement Email - Email Message Body
Enter body of message. HTML should NOT be used. Any links and line breaks will be auto converted on send. HTML tags will be stripped from the message if entered, so keep strictly to plain text and enter links without tags.

You can use the following tags to personalise the message:

{NAME} = Buyer Name
{EMAIL} = Buyer Email Address
{DATE} = Purchase Date
{INVOICE} = Invoice No
{CD} = CDs Purchased (if applicable)
{CDD} - CD Downloads Purchased (if applicable)
{TRACKS} - Track Downloads Purchased (if applicable)
{ID} - Transaction / Payment ID
If you wish to change the strings, open the 'global.php' language file and edit the '$emvars' array.
Other > License Agreement Email - Enable License Agreement Email
Enable or disable email.
Other > Maxmind GeoIP Database - Locate MaxMind IPv4 CSV File
Used to detect countries from IPv4 addresses. This is a free download from Maxmind. Note that you should download the zip file and unzip the CSV enclosed. DO NOT upload the zip file. You also need to make sure that your server can upload files of this size. If not your PHP.ini file may need adjusting and the server rebooting. Refer to the display for your limits. See the following screenshot:


At the time of these docs, the download location was:
http://geolite.maxmind.com/download/geoip/database/GeoIPCountryCSV.zip

Important: BOTH CSV files must be uploaded at any one time. DO NOT just upload one file.

Important: The MySQL function "LOAD DATA LOCAL INFILE" must be available in your MySQL version for this to work. If it isn`t the import will fail.
