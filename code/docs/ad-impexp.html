Import / Export

Overview
Enables quick importing of collections and tracks. Useful if you have a large catalogue and are exporting from other software. Before importing your CSV MUST meet the required criteria. Invalid data may cause the import to fail. Please see below for expected columns and example files.

NOTE: Very large import files may time out. It is recommended you break up large files into smaller chunks if your files contain more than 5000 rows. There are a number of free utilities online for splitting CSV files. If you find that your import hangs, try smaller files.
Import Collections
Your CSV doesn`t need to contain a header line. All rows are read, so including an header line could cause the import to fail. The CSV MUST contain the following columns of data in the order they are shown. Data separator MUST be a comma and the field enclosure / escape character is always a double quote. All data can be edited after the import.

The collection import expects 15 columns:

name of collection (max 250 chars)
title to appear in browser if applicable
meta keyword if applicable
meta description if applicable
rewrite url slug value if appliable (max 50 chars)
information about collection
search tags (comma delimited)
cover art path relative to cover art folder
hit counter (numeric value, max 10. set to 0 if start count is 0)
release date (numeric timestamp ONLY or 0)
catalogue number if applicable (max 200 chars)
cost of download (max 10 chars)
cost of cd (max 10 chars)
total length of collection (max 10 chars)
average bitrate of collection (max 100 chars)
Locate CSV = Use this to locate your CSV file.
Music Styles = Choose styles applicable to import
Clear ALL Existing Collections = If checked, all current collections are cleared before the import

Import Music Tracks
Your CSV doesn`t need to contain a header line. All rows are read, so including an header line could cause the import to fail. The CSV MUST contain the following columns of data in the order they are shown. Data separator MUST be a comma and the field enclosure / escape character is always a double quote. All data can be edited after the import.

The track import expects 8 columns:

title of music file (max 250 chars)
mp3 file path relevant to secure folder
preview mp3 file relative to the preview folder
length of track (max 10 chars)
bitrate (max 100 chars)
sample rate (max 100 chars)
cost of track (max 10 chars)
order (numeric value of order preference. if setting later, set this as 0)
Locate CSV = Use this to locate your CSV file.
Choose Collection = Choose collection applicable to import
Clear Existing Tracks for Collection = If checked, all current tracks are cleared for the selected collection before the import

Export Options
Exports data to CSV. All data from database is exported, with the exception of the ID column. The export will ONLY re-import if the file meets the above criteria. All data is included for export into other applications, so not necessarily back into Maian Music.
